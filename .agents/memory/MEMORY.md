- [OpenEmbedded color palette](openembedded-colors.md) — primary #1a1a1a, secondary/canvas #111111, elevated #222222, float #2a2a2a; zero saturation — NO purple/blurple anywhere; primary buttons are near-white (#efefef) on dark, not color.
- [OpenEmbedded UI conventions](openembedded-ui.md) — tab bars use underline indicator (neutral white on active, transparent inactive); DiscordPreview keeps Discord's own color palette (CHAT_BG #313338 etc); focus ring rgba(255,255,255,0.35) in index.css; no purple in any UI chrome.
- [OpenEmbedded interaction edge system](openembedded-interaction-edges.md) — dual edge type system: "default" = structural parent-child, "interaction" = on-click flow; compiler must filter edges by type !== "interaction" before building hierarchy; NodeWrapper nodeClass "interactive" + showInteractionHandle prop controls amber handle visibility.
- [OpenEmbedded Bot Node architecture](openbot-architecture.md) — openbot/ is a standalone Discord.js v14 + Express project deployed separately; api-server proxies via OPENBOT_API_URL + OPENBOT_API_KEY env vars; frontend uses generated useOpenBotGuilds/Channels/Send hooks; flows compiled and registered at send time.
- [OpenEmbedded security architecture](openembedded-security.md) — full security hardening: session auth (express-session+pg), owner_id on projects, Helmet headers, rate limits, CORS lockdown, strict webhook regex, bot token format validation.
- [OpenEmbedded web auth flow](openembedded-web-auth.md) — AuthProvider (authContext.tsx) + AuthGuard wrap protected routes; login page at /login, callback at /auth/callback; CSRF state in sessionStorage; code exchange server-side via POST /v1/auth/login with session fixation prevention.
