## 🏗️ PRODUCT OVERVIEW

**OpenEmbedded** is a visual, node-graph-based builder for Discord bot developers and server admins to:

- Visually construct Discord embeds and Components V2 messages using a node canvas
- Connect nodes (Text Display → Section → Container → Action Row) to define layout hierarchy
- Preview the output in a pixel-accurate Discord UI mock in real time
- Export the final payload as JSON, a discord.js code snippet, or raw API payload
- Send directly to a Discord webhook or bot endpoint

**Target users:** Discord bot developers, server admins , Discord Users and community

## 🔷 NODE SYSTEM

### Node Types Reference

Every node maps 1:1 to a Discord Components V2 component type:

| Node Name | Component Type | Inputs | Outputs |
|---|---|---|---|
| `TextDisplayNode` | type: 10 | — | content |
| `ThumbnailNode` | type: 11 | url, description | accessory |
| `MediaGalleryNode` | type: 12 | items[] | content |
| `FileNode` | type: 13 | file attachment | content |
| `SeparatorNode` | type: 14 | spacing, divider | content |
| `SectionNode` | type: 9 | text[], accessory | content |
| `ContainerNode` | type: 17 | content[], accent_color | root |
| `ActionRowNode` | type: 1 | components[] | content |
| `ButtonNode` | type: 2 | label, style, custom_id/url | component |
| `SelectMenuNode` | type: 3–8 | options, placeholder | component |
| `EmbedNode` | legacy embed | title, description, fields, color, etc. | root |

### Node Anatomy Rules

Every node component must:
1. Render inside a `NodeWrapper` component that handles selection border (blurple glow), drag, and resize
2. Display its type badge in top-left using `--color-text-faint` text on `--color-bg-surface` background
3. Show validation errors inline using `--color-danger` border + icon
4. Expose `data` via Zustand `graphStore` — never local state for anything that affects the payload
5. All handles: input handles on LEFT (green `#57F287`), output handles on RIGHT (blurple `#5865F2`)

### Node Data Interface Pattern

```ts
// packages/shared/src/types/nodes.ts
export interface BaseNodeData {
  id: string
  label: string
  componentType: number
  isValid: boolean
  errors: string[]
}

export interface TextDisplayNodeData extends BaseNodeData {
  componentType: 10

}

export interface ContainerNodeData extends BaseNodeData {
  componentType: 17
  accent_color?: number // integer color value
  spoiler?: boolean
}
// ... etc for each node type
```

```ts

  payload: DiscordMessagePayload | null
  isValid: boolean

```

The `compile()` function in `previewStore` calls `lib/compiler.ts` which traverses the graph (topological sort from root Container/Embed nodes) and produces the final Discord API payload object.

---

## ⚙️ COMPILER LOGIC (`lib/compiler.ts`)

The compiler must:

1. Find all **root nodes** (Container, Embed — nodes with no incoming edges or flagged as root)
2. Topologically sort child nodes via edge traversal
3. Recursively build the components array following Discord's nesting rules:
   - Container → children (Section, TextDisplay, Separator, MediaGallery, File, ActionRow)
   - Section → text (TextDisplay[]) + accessory (Thumbnail | Button)
   - ActionRow → components (Button[] | single Select)
4. Set `flags: 32768` (`IS_COMPONENTS_V2`) automatically when any V2 component is present
5. Return a typed `DiscordMessagePayload` object
6. Run Zod validation and surface errors per-node back to `graphStore`

## 🔌 BACKEND API ROUTES

### Base URL: `/api/v1`

```
GET    /projects              → List all saved projects
POST   /projects              → Create new project { name, graph: GraphJSON }
GET    /projects/:id          → Load project
PUT    /projects/:id          → Update project graph
DELETE /projects/:id          → Delete project

POST   /export/json           → Validate + return Discord payload JSON
POST   /export/code           → Return discord.js v14 code string
POST   /webhook/send          → Proxy send payload to Discord webhook URL
```

### Project Schema (PostgreSQL via Drizzle)

```ts
export const projects = pgTable('projects', {
  id:         uuid('id').defaultRandom().primaryKey(),
  name:       varchar('name', { length: 255 }).notNull(),
  graph:      jsonb('graph').notNull(),        // { nodes, edges }
  payload:    jsonb('payload'),                // last compiled payload
  createdAt:  timestamp('created_at').defaultNow(),
  updatedAt:  timestamp('updated_at').defaultNow(),
})
```
